# 汇报大纲：物联网技术在校园一卡通系统中的落地实践

## 1. 课题背景：为什么一卡通是典型的物联网？

校园一卡通不仅仅是一张塑料卡片，它是一个集**感知、传输、处理、应用**于一体的典型物联网系统。它解决了校园内身份识别（你是谁）和支付结算（你花了多少钱）的数字化连接问题。

## 2. 校园一卡通的体系架构（三层模型）

按照物联网的标准三层架构，一卡通系统可以完美对应：

### 2.1 感知识别层 (Perception Layer) —— “系统的触角”

* **核心技术：** RFID (无线射频识别)、NFC、二维码、人脸识别。
* **硬件终端：** IC卡： 存储用户UID和加密扇区数据。
* **读卡器/POS机：** 通过电磁感应读取卡片信息。
* **传感器：** 水控机中的流量传感器、电控中的电流互感器。

### 2.2 网络传输层 (Network Layer) —— “系统的神经”

* **通信协议：** TCP/IP (以太网)、RS-485总线、Wi-Fi、ZigBee (常用于低功耗宿舍水表)。
* **组网方式：** 校园骨干网专网隔离，确保交易数据不与外网混用，提高安全性。

### 2.3 应用支撑层 (Application Layer) —— “系统的大脑”

* **后端服务：** 基于 Spring Boot 等框架处理业务逻辑。
* **数据中心：** 存储用户信息、卡片状态、消费流水。
* **业务模块：** 支付结算、图书馆借阅管理、宿舍门禁权限、机房考勤。

## 3. 核心技术点解析

| 技术维度 | 关键点 | 物联网体现 |
| --- | --- | --- |
| **射频通信** | ISO/IEC 14443 协议 (M1卡) | 非接触式近场数据交换 |
| **嵌入式开发** | 单片机 (STM32/ESP32) 驱动读卡模块 | 终端设备的边缘智能 |
| **数据安全** | DES/AES 对称加密、扇区校验 | 保证物联网节点间传输的数据不可篡改 |
| **并发处理** | 数据库事务控制 ($Transaction$) | 保证高频刷卡时“余额”与“流水”的强一致性 |

## 4. 开源案例讲解：`yanweijia/cardSystem` 深度剖析

结合 GitHub 项目 [yanweijia/cardSystem](https://github.com/yanweijia/cardSystem)，我们可以看到物联网应用层是如何实现的。

### 4.1 核心实体映射 (Physical to Digital)

在代码中，**“物理卡片”**被抽象为了数据库模型：

* `User` 实体：对应持卡人生物信息。
* `Card` 实体：对应 RFID 卡片的逻辑表示（包含余额、状态等）。
* `Bill` 实体：对应物联网感应层触发的一次“事件”。

### 4.2 业务逻辑流程 (以消费为例)

1. **感应触发：** 模拟终端向后端接口发送一个消费请求 `POST /consume`。
2. **原子性操作：** 在 `CardService` 中，系统会执行一个核心数学公式：

$$Balance_{new} = Balance_{old} - Amount_{spend}$$


3. **状态反馈：** 后端处理完毕后返回指令，读卡器（感知层）接收到成功信号，发出“滴”声并弹开电磁锁（执行层）。

### 4.3 该项目的学习价值

* **数据可视化：** 该项目包含了消费统计图表，这对应了物联网中的**数据挖掘**——通过分析学生的刷卡频次，可以推断食堂的拥挤时段或学生的消费水平。
